Request Data Flow
1
User Interaction
User selects filters (Period, Location, Category, Status) in FilterBar.vue or navigates to a view.
2
Global Filter State
useFilters.js composable updates reactive refs shared across all view components.
selectedPeriod · selectedLocation · selectedCategory · selectedStatus
3
API Call via api.js
Component watches filter changes; calls the appropriate method in api.js, which builds query params from filter state.
GET /api/orders?warehouse=Tokyo&month=2025-03
4
FastAPI Endpoint
Receives query params. Calls apply_filters() and filter_by_month() helpers on the in-memory dataset.
5
In-Memory Filter
Python list comprehensions filter the JSON data that was loaded from disk at server startup. No database queries.
6
Pydantic Serialisation
Response models validate and serialize the filtered list to JSON. FastAPI sends the response back to the browser.
7
Vue Renders
Component stores data in a ref. Computed properties transform it; template renders with v-for and conditional logic.

Key Design Decisions
No database — data lives in JSON files loaded once at startup; all filtering is in-memory Python. Simplifies setup for workshop use.
Singleton composables — useFilters is a module-level singleton, so filter state is shared across all Vue components without Vuex/Pinia.
CORS open — backend allows all origins (*) so the Vite dev server can call the API without proxy configuration.
i18n built-in — English and Japanese supported via a lightweight custom composable; no external i18n library required.
OpenAPI docs — FastAPI auto-generates interactive docs at /docs, useful for exploring endpoints during development.
