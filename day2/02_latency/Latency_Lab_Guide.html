Inference Optimisation.

Benchmark streaming latency, compare the Claude model family, measure tool-call overhead, and cut costs with prompt caching — all with real API calls.

01

Track metrics

Build BenchmarkResult, _stream_request, compute_otps, calculate_cost.
Instrument every streaming call to capture TTFT (first event), generation time, OTPS, and dollar cost. Store results in a typed dataclass so the BenchmarkSuite can aggregate and tabulate them.
02

Benchmark the model family

Run Haiku 4.5, Sonnet 4.5, Opus 4.5 on the same prompt — 5 runs each.
Use measure_streaming_latency to collect TTFT, TTC, OTPS, and cost for each model. The BenchmarkSuite.summary() prints a ranked grid so you can see the speed/quality/cost trade-off at a glance.
03

Add complexity: tool use

Define a calculator tool and measure the two-call round-trip overhead.
Call 1 → model returns tool_use block → execute locally → Call 2 → model returns final text. Compare TTC with and without the tool. TTFT resets each turn — always use TTC for agentic flows.
04

Add countermeasures: prompt caching

Mark the system prompt with cache_control: ephemeral and track cold vs. warm latency.
A large static system prompt (>1024 tokens) is re-processed every request. After caching, subsequent calls read from the KV store — dramatically cutting TTFT and input-token cost. Also implemented for multi-turn conversations.
01

The Four Latency Metrics

Captured on every streaming call via _stream_request().

TTFT
Time to First Token
How long before the model starts responding. Triggered by the content_block_start event in the stream.
t_first_event − t_start
OTPS
Output Tokens Per Second
Streaming throughput after the first token. Computed from gen_time = TTC − TTFT.
output_tokens / gen_time
TTC
Time to Completion
Total wall-clock time including stream close. Primary metric for tool-use flows where TTFT resets per turn.
t_after_stream − t_start
Cost
Request Cost ($)
Computed per request from input/output token counts and per-model pricing. Aggregated by BenchmarkSuite as $/1K calls.
(in_tok/1M)×$in + (out_tok/1M)×$out

# _stream_request — captures TTFT via content_block_start event
def _stream_request(prompt, model=DEFAULT_MODEL, max_tokens=256):
    ttft = None
    start_time = time.perf_counter()
    with client.messages.stream(
        model=model, max_tokens=max_tokens,
        messages=[{"role": "user", "content": prompt}]
    ) as stream:
        for event in stream:
            if ttft is None and event.type == "content_block_start":
                ttft = time.perf_counter() - start_time   # ← TTFT here
        response = stream.get_final_message()
    total_time = time.perf_counter() - start_time         # ← TTC here
    return ttft, total_time, response

# compute_otps — generation time excludes TTFT (server think time)
def compute_otps(ttft, total_time, output_tokens):
    gen_time = total_time - ttft                          # time actually streaming
    otps = output_tokens / gen_time if gen_time > 0 else 0.0
    return otps, gen_time

# calculate_cost — per-model pricing ($/1M tokens)
PRICING = {
    "claude-haiku-4-5-20251001":  {"input": 0.25,  "output": 1.25},
    "claude-sonnet-4-5-20250929": {"input": 3.00,  "output": 15.00},
    "claude-opus-4-5-20251101":   {"input": 15.00, "output": 75.00},
}

def calculate_cost(model, input_tokens, output_tokens):
    prices = PRICING.get(model, {"input": 3.00, "output": 15.00})
    input_cost  = (input_tokens  / 1_000_000) * prices["input"]
    output_cost = (output_tokens / 1_000_000) * prices["output"]
    return input_cost, output_cost, input_cost + output_cost

# measure_tool_use_latency — full 2-call round-trip
start_time = time.perf_counter()

first_response = client.messages.create(
    model=model, max_tokens=max_tokens, tools=[CALCULATOR_TOOL],
    messages=[{"role": "user", "content": prompt}],
)
ttft = time.perf_counter() - start_time    # ← TTFT after Call 1

tool_use_block = next(
    (b for b in first_response.content if b.type == "tool_use"), None
)
result = execute_calculator(tool_use_block.input["operation"], tool_use_block.input["operands"])

second_response = client.messages.create(
    model=model, max_tokens=max_tokens, tools=[CALCULATOR_TOOL],
    messages=[
        {"role": "user",      "content": prompt},
        {"role": "assistant", "content": first_response.content},
        {"role": "user",      "content": [{
            "type": "tool_result", "tool_use_id": tool_use_block.id,
            "content": str(result)
        }]},
    ],
)
total_time = time.perf_counter() - start_time  # ← TTC after Call 2

# system_block — static system prompt marked for caching
# Must be >1024 tokens for cache to activate
SYSTEM_PROMPT = """You are an expert API documentation assistant...""" * 20

system_block = [{
    "type": "text",
    "text": SYSTEM_PROMPT,
    "cache_control": {"type": "ephemeral"}   # ← breakpoint here
}]

def cached_request(question):
    start = time.perf_counter()
    response = client.messages.create(
        model=DEFAULT_MODEL, max_tokens=256,
        system=system_block,
        messages=[{"role": "user", "content": question}],
    )
    elapsed = time.perf_counter() - start
    return response, elapsed

# chat() — multi-turn caching: mark last assistant msg before each call
def chat(messages, new_question):
    # Strip old cache breakpoints from earlier turns
    for msg in messages:
        if msg["role"] == "assistant" and isinstance(msg["content"], list):
            msg["content"] = msg["content"][0]["text"]

    # Mark the last assistant message — cache everything up to here
    if messages and messages[-1]["role"] == "assistant":
        last = messages[-1]
        text = last["content"] if isinstance(last["content"], str) else last["content"][0]["text"]
        last["content"] = [{
            "type": "text", "text": text,
            "cache_control": {"type": "ephemeral"}   # ← grows each turn
        }]

    messages.append({"role": "user", "content": new_question})
    response = client.messages.create(model=MODEL_SONNET, max_tokens=300,
                                       system=SYSTEM, messages=messages)
    messages.append({"role": "assistant", "content": response.content[0].text})
    return response.content[0].text, response.usage

Cache these (static)

  • System prompt / persona
  • Tool definitions
  • Long document / knowledge base
  • Few-shot examples
  • Previous assistant turns in a conversation

Don't cache these (dynamic)

  • User name / session ID injected into system prompt
  • Current date/time before the breakpoint
  • Per-request variables in the prefix
  • Anything that changes between calls
Cache rules: TTL is 5 minutes. Minimum 1024 tokens to activate. Any change to content before the breakpoint invalidates the cache. Put dynamic content after the last cache_control marker.
